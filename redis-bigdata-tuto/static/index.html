    • Why Redis?

    In Big Data systems, we need fast access to frequently-accessed data. Redis provides sub-millisecond latency by storing data in RAM instead of disk.

    Key Concepts:
    • Cache Hit - Data found in Redis (fast, ~1-2ms)
    • Cache Miss - Data not in Redis, must query source (slow, ~50ms)
    • TTL - Time To Live: when data expires and is removed
    • Memory - Redis uses RAM, which is limited and expensive
